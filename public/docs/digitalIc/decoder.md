## Decoder (ဒစ်ဂျစ်တယ် ကုဒ်ဖော်ကိရိယာ)

### ခြုံငုံသုံးသပ်ချက် (Overview)
Decoder ဆိုသည်မှာ ဒစ်ဂျစ်တယ်စနစ်များမှ ပေးပို့လိုက်သော Binary Code (ကုဒ်) များကို မူလသတင်းအချက်အလက်များအဖြစ် ပြန်လည်ဖော်ထုတ်ပေးသည့် ပတ်လမ်းတစ်ခု ဖြစ်ပါသည်။ ၎င်းသည် Encoder ၏ ပြောင်းပြန် အလုပ်လုပ်သော ကိရိယာ ဖြစ်သည်။ Decoder တစ်ခုတွင် အဝင်လမ်းကြောင်း (Input) $n$ ခု ရှိပါက အထွက်လမ်းကြောင်း (Output) $2^n$ ခု ရှိမည် ဖြစ်သည်။ အဝင်တွင် သတ်မှတ်ထားသော binary code တစ်ခု ပေးလိုက်ပါက ၎င်းနှင့် သက်ဆိုင်သော အထွက်လမ်းကြောင်း တစ်ခုတည်းသာလျှင် အလုပ်လုပ် (Active ဖြစ်) မည် ဖြစ်သည်။

---

### သမိုင်းကြောင်း (History)
Decoder များသည် ၁၉၄၀ နှင့် ၅၀ ပြည့်နှစ်များရှိ အစောပိုင်း ကွန်ပျူတာများ၏ မှတ်ဉာဏ်စနစ် (Memory Addressing) အတွက် အလွန်အရေးပါခဲ့သည်။ ၁၉၆၀ နှင့် ၇၀ ပြည့်နှစ်များတွင် **74138** (3-to-8 Decoder) နှင့် **7447** (BCD-to-7-Segment Decoder) ကဲ့သို့သော IC များ ပေါ်ထွက်လာခဲ့သည်။ ဤ IC များကြောင့် ဒစ်ဂျစ်တယ် ဂဏန်းပြဘုတ်များကို လွယ်ကူစွာ ထိန်းချုပ်နိုင်လာခဲ့ပြီး ကွန်ပျူတာ CPU အတွင်းရှိ အစိတ်အပိုင်းများကို တိကျစွာ ရွေးချယ်ခိုင်းစေနိုင်သည့် စနစ်များ ပေါ်ပေါက်လာခဲ့သည်။

---

### အလုပ်လုပ်ပုံ (Working Principle)
Decoder သည် ကုဒ်များကို ခွဲခြားမှတ်သားခြင်း (Code Recognition) စနစ်ဖြင့် အလုပ်လုပ်သည် -

* **ကုဒ်ဖော်ခြင်း-** အဝင်တွင် ဝင်လာသော 0 နှင့် 1 ပေါင်းစပ်မှုများကို ကြည့်ရှုသည်။ အဝင် $n$ ခု ရှိလျှင် ဖြစ်နိုင်ခြေရှိသော အခြေအနေ $2^n$ ခု ရှိသည်။
* **သီးသန့်အထွက်-** အဝင်ကုဒ်တစ်ခုစီအတွက် သီးသန့်အထွက်လမ်းကြောင်း တစ်ခုစီ ရှိသည်။ ဥပမာ - 2-to-4 decoder တစ်ခုတွင် အဝင်ကို 01 (Binary တန်ဖိုး ၁) ပေးပါက အထွက်လမ်းကြောင်း နံပါတ် ၁ တစ်ခုတည်းသာ အလုပ်လုပ်မည် ဖြစ်သည်။
* **Enable Pin:** Decoder IC အများစုတွင် Enable pin ပါရှိသည်။ အကယ်၍ Enable pin ကို မဖွင့်ထားပါက အဝင်တွင် မည်သည့်ကုဒ်ပေးပေး အထွက်တွင် မည်သည့်လမ်းကြောင်းမျှ အလုပ်လုပ်မည် မဟုတ်ပါ။

---

### Decoder အမျိုးအစားများ
1. **Binary Decoder:** အဝင် $n$ ကို အထွက် $2^n$ သို့ ပြောင်းပေးခြင်း (ဥပမာ - 3-to-8 decoder)။
2. **BCD-to-Decimal Decoder:** 4-bit BCD ကုဒ်ကို ၀ မှ ၉ အထိ လမ်းကြောင်း ၁၀ ခုအဖြစ် ပြောင်းပေးခြင်း။
3. **BCD-to-7-Segment Decoder:** Binary ဒေတာများကို 7-segment display ပေါ်တွင် ဂဏန်းများအဖြစ် မြင်ရအောင် လိုအပ်သော ငုတ်များကို လင်းစေသည့် IC (ဥပမာ - **7447, 7448 IC**)။
4. **Address Decoder:** ကွန်ပျူတာ CPU က မည်သည့် Memory chip ကို သုံးမည်ဆိုသည်ကို ရွေးချယ်ရန် သုံးသည့် သီးသန့် Decoder။

---

### အသုံးချပုံများ (Applications)
* **7-Segment Displays:** ဒစ်ဂျစ်တယ်နာရီများ၊ ဂဏန်းတွက်စက်များနှင့် မီတာများတွင် ဂဏန်းများ ပြသရန် သုံးခြင်း။
* **Memory Addressing:** RAM chip များအတွင်းရှိ အချက်အလက် သိမ်းထားသော နေရာအတိအကျကို ရွေးချယ်ရန် သုံးခြင်း။
* **လမ်းကြောင်းခွဲထုတ်ခြင်း (Demultiplexing):** လာသမျှ Data များကို လိုအပ်သော လမ်းကြောင်းအသီးသီးသို့ ခွဲဝေပို့ဆောင်ခြင်း။
* **Instruction Decoding:** CPU က လက်ခံရရှိသော အမိန့်ကုဒ်များကို ကြည့်ပြီး မည်သည့်အလုပ် (ပေါင်းခြင်း၊ နှုတ်ခြင်း) လုပ်ရမည်ကို ခွဲခြားသတ်မှတ်ခြင်း။
