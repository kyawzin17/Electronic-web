## Multiplexer (ဒစ်ဂျစ်တယ် လမ်းကြောင်းရွေးချယ်ကိရိယာ)

### ခြုံငုံသုံးသပ်ချက် (Overview)
Multiplexer (အတိုကောက် **MUX**) ဆိုသည်မှာ အဝင်လမ်းကြောင်း (Input lines) အများအပြားမှ လာသော အချက်အလက်များထဲမှ တစ်ခုကို ရွေးချယ်ပြီး အထွက်လမ်းကြောင်း (Output) တစ်ခုတည်းသို့ ပို့ဆောင်ပေးသည့် ပတ်လမ်းတစ်ခု ဖြစ်ပါသည်။ ၎င်းကို ဒစ်ဂျစ်တယ်စနစ်သုံး "**လမ်းကြောင်းရွေးချယ်သည့် ခလုတ်**" ဟု တင်စားနိုင်ပါသည်။ MUX တစ်ခုတွင် အဝင်လမ်းကြောင်း $2^n$ ခု ရှိပါက ရွေးချယ်မှုပြုလုပ်သည့် Select line $n$ ခုနှင့် အထွက်လမ်းကြောင်း ၁ ခု ပါရှိမည် ဖြစ်သည်။ Select line ပေါ်တွင် ပေးလိုက်သော ကုဒ်ပေါ်မူတည်၍ မည်သည့်အဝင်လမ်းကြောင်းမှ Data များ အပြင်သို့ ထွက်ရမည်ကို ဆုံးဖြတ်ပေးခြင်း ဖြစ်သည်။

### သမိုင်းကြောင်း (History)
Multiplexing အယူအဆသည် ၁၈၇၀ ပြည့်နှစ်များရှိ ကြေးနန်းဆက်သွယ်ရေး ခေတ်ကတည်းက ကြိုးတစ်ပင်တည်းမှ သတင်းစကားများစွာ ပေးပို့နိုင်ရန် စတင်ခဲ့ခြင်း ဖြစ်သည်။ ၁၉၆၀ ပြည့်နှစ်များတွင် **74151** နှင့် **74153** ကဲ့သို့သော MUX IC များ ပေါ်ထွက်လာခဲ့သည်။ ဤ IC များကြောင့် ကွန်ပျူတာ CPU အတွင်းရှိ ဒေတာများကို လိုအပ်သလို လမ်းကြောင်းပြောင်းလဲပေးခြင်း (**Data Routing**) ကို လွယ်ကူစွာ လုပ်ဆောင်နိုင်လာခဲ့ပြီး ပတ်လမ်းအတွင်း ရှုပ်ထွေးလှသော ကြိုးလိုင်းများကို များစွာ လျှော့ချပေးနိုင်ခဲ့သည်။

---

### အလုပ်လုပ်ပုံ (Working Principle)
Multiplexer သည် အချက်အလက် ရွေးချယ်ခြင်း (**Data Selection**) စနစ်ဖြင့် အလုပ်လုပ်သည် -

* **ရွေးချယ်မှုယုတ္တိ-** "Select" ငုတ်များတွင် ပေးလိုက်သော Binary code သည် လိပ်စာ (Address) တစ်ခုကဲ့သို့ အလုပ်လုပ်သည်။
* **လမ်းကြောင်းပြောင်းလဲခြင်း-** ဥပမာ - 4-to-1 MUX တစ်ခုတွင် Select line များကို 10 (Binary တန်ဖိုး ၂) ပေးလိုက်ပါက အဝင်လမ်းကြောင်း နံပါတ် ၂ မှ Data များသည် အထွက်သို့ စီးဆင်းသွားမည် ဖြစ်ပြီး ကျန်ရှိသော အဝင်များကို ပိတ်ထားမည် ဖြစ်သည်။
* **ပေါင်းစည်းခြင်း-** ၎င်းသည် လမ်းကြောင်းများစွာမှ ဒေတာများကို လမ်းကြောင်းတစ်ခုတည်းသို့ စုစည်းပေးခြင်း ဖြစ်သည်။

---

### Multiplexer အမျိုးအစားများ
1. **2-to-1 Multiplexer:** အဝင် ၂ ခု၊ Select line ၁ ခု၊ အထွက် ၁ ခု။
2. **4-to-1 Multiplexer:** အဝင် ၄ ခု၊ Select line ၂ ခု၊ အထွက် ၁ ခု။
3. **8-to-1 Multiplexer:** အဝင် ၈ ခု၊ Select line ၃ ခု၊ အထွက် ၁ ခု (ဥပမာ - **74LS151 IC**)။
4. **Analog Multiplexer:** ဒစ်ဂျစ်တယ် Signal များသာမက အသံ (Audio) နှင့် Sensor ဗို့အားများကဲ့သို့ Analog signal များကိုပါ လမ်းကြောင်းရွေးချယ်ပေးနိုင်သည့် အမျိုးအစား (ဥပမာ - **CD4051 IC**)။

---

### အသုံးချပုံများ (Applications)
* **ဆက်သွယ်ရေးစနစ်များ-** ဖိုင်ဘာအော်ပတစ် ကြိုးတစ်ပင်တည်းမှ ဖုန်းပြောဆိုမှု သို့မဟုတ် ဒေတာများစွာကို တစ်ပြိုင်နက် ပေးပို့ခြင်း။
* **ဒေတာလမ်းကြောင်းပြောင်းခြင်း-** ကွန်ပျူတာ ပရိုဆက်ဆာထံသို့ မည်သည့်နေရာ (Memory သို့မဟုတ် Sensor) မှ Data များ ဝင်လာရမည်ကို ရွေးချယ်ပေးခြင်း။
* **Parallel-to-Serial Conversion:** အပြိုင်ဝင်လာသော Data များကို တစ်ခုချင်းစီ အစီအစဉ်လိုက် လမ်းကြောင်းတစ်ခုတည်းမှ ထွက်သွားအောင် ပြုလုပ်ခြင်း။
* **Signal Multiplexing:** Microcontroller တစ်ခုတွင် ADC ငုတ် တစ်ခုတည်းဖြင့် Sensor ပေါင်းများစွာကို တစ်လှည့်စီ ဖတ်ရှုရာတွင် သုံးခြင်း။

---

### အခြေခံပုံသေနည်း
အဝင်လမ်းကြောင်း ($I$) နှင့် Select line ($S$) တို့၏ ဆက်သွယ်ချက်မှာ -

&emsp; **$$I = 2^S$$**

> *(ဥပမာ - Select line ၃ ခု ရှိပါက အဝင်လမ်းကြောင်း $2^3 = 8$ ခုကို ထိန်းချုပ်နိုင်သည်)*
