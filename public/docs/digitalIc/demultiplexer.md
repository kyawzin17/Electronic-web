## Demultiplexer (ဒစ်ဂျစ်တယ် အချက်ပြခွဲဝေကိရိယာ)

### ခြုံငုံသုံးသပ်ချက် (Overview)
Demultiplexer (အတိုကောက် **DEMUX**) ဆိုသည်မှာ အဝင်လမ်းကြောင်း (Input) တစ်ခုတည်းမှ လာသော အချက်အလက်များကို အထွက်လမ်းကြောင်း (Output lines) အမြောက်အမြားထဲမှ လိုအပ်သော လမ်းကြောင်းတစ်ခုဆီသို့ ရွေးချယ်ပို့ဆောင်ပေးသည့် ပတ်လမ်းတစ်ခု ဖြစ်ပါသည်။ ၎င်းသည် Multiplexer ၏ ပြောင်းပြန် အလုပ်လုပ်သော ကိရိယာ ဖြစ်သည်။ DEMUX တစ်ခုတွင် Data အဝင်လမ်းကြောင်း ၁ ခု၊ ရွေးချယ်မှုပြုလုပ်သည့် Select line $n$ ခုနှင့် အထွက်လမ်းကြောင်း $2^n$ ခု ပါရှိမည် ဖြစ်သည်။ Select line ပေါ်ရှိ ကုဒ်ကို ကြည့်ပြီး အဝင်မှ Data များကို မည်သည့်အထွက်လမ်းကြောင်းသို့ ပို့ပေးရမည်ကို ဆုံးဖြတ်ခြင်း ဖြစ်သည်။

---

### သမိုင်းကြောင်း (History)
Demultiplexing အယူအဆသည် ၂၀ ရာစုအလယ်ပိုင်း ဖုန်းလိုင်းဆက်သွယ်ရေးစနစ်များ (Telephone exchange) ကျယ်ပြန့်လာချိန်တွင် အလွန်အရေးပါလာခဲ့သည်။ ၁၉၆၀ နှင့် ၇၀ ပြည့်နှစ်များတွင် ဒစ်ဂျစ်တယ်ဆက်သွယ်ရေးစနစ်များ ထွန်းကားလာသောအခါ **74138** နှင့် **74154** ကဲ့သို့သော DEMUX IC များ ပေါ်ထွက်လာခဲ့သည်။ ဤ IC များကြောင့် အချက်ပြလှိုင်း ပေးပို့သည့် လမ်းကြောင်းတစ်ခုတည်းမှ ဒေတာများကို အသုံးပြုသူများစွာ သို့မဟုတ် စက်ပစ္စည်းများစွာထံသို့ အမြန်နှုန်းဖြင့် ခွဲဝေပို့ဆောင်နိုင်လာခဲ့သည်။

---

### အလုပ်လုပ်ပုံ (Working Principle)
Demultiplexer သည် အချက်အလက် ခွဲဝေခြင်း (**Data Distribution**) စနစ်ဖြင့် အလုပ်လုပ်သည် -

* **ရွေးချယ်မှုယုတ္တိ-** "Select" ငုတ်များသည် လမ်းညွှန်မြေပုံ သို့မဟုတ် ခလုတ်ထိန်းချုပ်သူကဲ့သို့ အလုပ်လုပ်သည်။
* **လမ်းကြောင်းခွဲထုတ်ခြင်း-** ဥပမာ - 1-to-4 DEMUX တစ်ခုတွင် Select line များကို 01 (Binary တန်ဖိုး ၁) ပေးလိုက်ပါက အဝင်မှ Data များသည် အထွက်လမ်းကြောင်း နံပါတ် ၁ ဆီသို့ စီးဆင်းသွားမည် ဖြစ်ပြီး ကျန်ရှိသော အထွက်လမ်းကြောင်းများကို ပိတ်ထားမည် ဖြစ်သည်။
* **တစ်ခုမှအများသို့-** ၎င်းကို ရထားလမ်းဆုံရှိ ရထားတစ်စင်းတည်းကို လိုအပ်သော လမ်းခွဲထဲသို့ ရောက်အောင် လမ်းကြောင်းပြောင်းပေးသည့် စနစ်နှင့် တင်စားနိုင်သည်။

---

### Demultiplexer အမျိုးအစားများ
1. **1-to-2 Demultiplexer:** အဝင် ၁ ခု၊ Select line ၁ ခု၊ အထွက် ၂ ခု။
2. **1-to-4 Demultiplexer:** အဝင် ၁ ခု၊ Select line ၂ ခု၊ အထွက် ၄ ခု။
3. **1-to-8 Demultiplexer:** အဝင် ၁ ခု၊ Select line ၃ ခု၊ အထွက် ၈ ခု (ဥပမာ - **74LS138 IC**)။
4. **Analog Demultiplexer:** အသံ (Audio) ကဲ့သို့ Analog signal များကို မူလအရည်အသွေး မပျက်ဘဲ လိုအပ်သော လမ်းကြောင်းများသို့ ခွဲဝေပေးနိုင်သည့် အမျိုးအစား (ဥပမာ - **CD4051 IC** ကိုလည်း DEMUX အဖြစ် သုံးနိုင်သည်)။

---

### အသုံးချပုံများ (Applications)
* **ဆက်သွယ်ရေးစနစ်များ-** ဖိုက်ဘာကြိုး သို့မဟုတ် ဂြိုဟ်တုမှလာသော Single stream data များကို သက်ဆိုင်ရာ ဖုန်းလိုင်းများ သို့မဟုတ် အသုံးပြုသူများထံ ခွဲဝေပေးခြင်း။
* **Serial-to-Parallel Conversion:** တစ်ခုချင်းစီလာသော Data bits များကို လမ်းကြောင်းခွဲပြီး သက်ဆိုင်ရာ Flip-flop များထဲသို့ ပို့ဆောင်ကာ အပြိုင် (Parallel) data အဖြစ် ပြောင်းလဲခြင်း။
* **Address Decoding:** ကွန်ပျူတာ၏ မှတ်ဉာဏ်စနစ် (Memory) တွင် CPU ပေးလိုက်သော Address အပေါ်မူတည်၍ သတ်မှတ်ထားသော Memory bank ကို ရွေးချယ်ပေးခြင်း။
* **အချက်ပြလမ်းကြောင်း ခွဲဝေခြင်း-** အသံလွှင့်စနစ်တစ်ခုတွင် အသံရင်းမြစ်တစ်ခုတည်းကို လိုအပ်သော စပီကာများ သို့မဟုတ် Channel များဆီသို့ ခွဲထုတ်ပေးခြင်း။

---

### အခြေခံဆက်သွယ်ချက်
Select line ($S$) နှင့် အထွက်လမ်းကြောင်း အရေအတွက် ($O$) တို့၏ ဆက်သွယ်ချက်မှာ -

&emsp; **$$O = 2^S$$**

> *(ဥပမာ - Select line ၄ ခု ရှိပါက အထွက်လမ်းကြောင်း $2^4 = 16$ ခုအထိ ခွဲထုတ်နိုင်သည်)*
