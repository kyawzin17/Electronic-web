## Latch (ဒစ်ဂျစ်တယ် အချက်အလက် ဆွဲကိုင်ထိန်းညှိစနစ်)

### ခြုံငုံသုံးသပ်ချက် (Overview)
Latch ဆိုသည်မှာ ဒစ်ဂျစ်တယ် အချက်အလက်များကို သိမ်းဆည်းထားနိုင်သော အခြေခံအကျဆုံး အစိတ်အပိုင်းတစ်ခု ဖြစ်ပါသည်။ ၎င်းသည် **Level-triggered** အမျိုးအစား ဖြစ်သောကြောင့် Control signal (Enable) ပွင့်နေသရွေ့ အဝင် (Input) ပြောင်းလဲသည်နှင့် အထွက် (Output) ပါ ချက်ချင်း လိုက်ပြောင်းပါသည်။ Latch များသည် Flip-Flop များ၏ အခြေခံမူလ ပုံစံများဖြစ်ပြီး ဒစ်ဂျစ်တယ် စနစ်များတွင် အချက်အလက်များကို ခေတ္တခဏ မြဲမြံစွာ ဆွဲကိုင်ထားရန် အသုံးပြုသည်။

---

### သမိုင်းကြောင်း (History)
Latch အယူအဆသည် ၁၉၃၀ နှင့် ၄၀ ပြည့်နှစ်များရှိ အစောပိုင်း Relay စနစ်သုံး ကွန်ပျူတာများကတည်းက စတင်ခဲ့သည်။ ထရန်စစ္စတာများ ပေါ်ပေါက်လာသောအခါ NOR သို့မဟုတ် NAND gate များကို အပြန်အလှန် ချိတ်ဆက်ထားသည့် **SR Latch** များသည် ၁-ဘစ် မှတ်ဉာဏ် (1-bit memory) တည်ဆောက်ရန်အတွက် စံနှုန်းတစ်ခု ဖြစ်လာခဲ့သည်။ ခေတ်ပေါ် CPU များတွင် Clock နှင့် အလုပ်လုပ်သော Flip-Flop များကို ပိုသုံးသော်လည်း၊ ရိုးရှင်းသော ဒေတာသိမ်းဆည်းမှုများနှင့် ခလုတ်များ၏ ဆူညံမှုကို ဖယ်ရှားသည့်နေရာများတွင် Latch ကို ယနေ့တိုင် သုံးစွဲနေဆဲ ဖြစ်သည်။

---

### အလုပ်လုပ်ပုံ (Working Principle)
Latch သည် တုံ့ပြန်မှုစနစ် (**Feedback**) ကို အသုံးပြု၍ အလုပ်လုပ်သည် -

[Image showing Level Triggering vs Edge Triggering waveforms]

* **အပြန်အလှန်ချိတ်ဆက်မှု (Cross-Coupling):** Gate တစ်ခုမှ ထွက်လာသော Output ကို နောက်တစ်ခု၏ Input ထဲသို့ ပြန်ထည့်ပေးထားသည်။ ဤနည်းဖြင့် ပတ်လမ်းသည် "၁" သို့မဟုတ် "၀" အခြေအနေတစ်ခုမှာ ပိတ်မိ (**Lock**) သွားခြင်း ဖြစ်သည်။
* **ဗို့အားအဆင့်ဖြင့် အလုပ်လုပ်ခြင်း (Level Triggering):** ၎င်းသည် Flip-Flop ကဲ့သို့ Clock edge ကို စောင့်စရာမလိုဘဲ Enable လိုင်း ပွင့်နေချိန်တွင် အဝင်ရှိ ပြောင်းလဲမှုတိုင်းကို အထွက်သို့ ချက်ချင်း ပို့ပေးသည်။
* **မှတ်သားထားခြင်း-** Enable လိုင်း ပိတ်သွားသည်နှင့် Latch သည် ၎င်း၏ နောက်ဆုံးရရှိခဲ့သော အခြေအနေကို လျှပ်စစ်ရှိနေသရွေ့ ဆက်လက် မှတ်သားထားမည် ဖြစ်သည်။

---

### အသုံးများသော Latch အမျိုးအစားများ
1. **SR Latch:** အခြေခံအကျဆုံး အမျိုးအစား။ Set ငုတ်က ၁ ဖြစ်စေပြီး Reset ငုတ်က ၀ ဖြစ်စေသည်။
2. **Gated SR Latch:** Enable ငုတ်တစ်ခု ထပ်တိုးထားပြီး ၎င်းငုတ် High ဖြစ်မှသာ S နှင့် R ကို လက်ခံသည်။
3. **D Latch (Transparent Latch):** အဝင်ရှိ Data ကို Enable ပွင့်နေချိန်တွင် အထွက်သို့ ပို့ပေးပြီး၊ Enable ပိတ်သွားချိန်တွင် ထိုဒေတာကို ဆွဲကိုင် (**Latch**) ထားပေးသည်။

### အသုံးချပုံများ (Applications)
1. **Switch Debouncing:** ခလုတ်များနှိပ်သည့်အခါ ထွက်ပေါ်လာသော မတည်ငြိမ်သည့် လျှပ်စစ်လှိုင်း (**Mechanical chatter**) များကို ဖယ်ရှားပြီး သန့်စင်သော Signal တစ်ခု ရရှိရန် သုံးသည်။
2. **ဒေတာ ခေတ္တသိမ်းဆည်းခြင်း-** ပတ်လမ်းတစ်ခု၏ အစိတ်အပိုင်းတစ်ခုမှ နောက်တစ်ခုသို့ အချက်အလက်များ မလွှဲပြောင်းမီ ခေတ္တ ဆွဲကိုင်ထားရန် သုံးသည်။
3. **I/O Buffering:** Display များ သို့မဟုတ် မော်တာထိန်းချုပ်စနစ်များက အချက်အလက်ကို တည်ငြိမ်စွာ ဖတ်နိုင်အောင် ပြုလုပ်ပေးခြင်း။
4. **Asynchronous Systems:** နာရီချက်လှိုင်း (Clock) မရှိသော ပတ်လမ်းများတွင် အခြေအနေများကို မှတ်သားရန် သုံးသည်။
